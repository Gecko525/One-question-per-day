get和post是http协议的两种请求方式，而http是基于tcp/ip的应用层协议。所以用的都是同一个传输协议，在本质上，没有区别。 两者的区别主要表现在客户端。 1.可见性{ get：数据暴露在url。 post: 数据不显示在url。 } 2.浏览器记录{ get：参数保存在浏览器记录。 post: 参数不保存在浏览器记录。 } 3.缓存{ get：能被缓存。 post: 不能被缓存。 } 4.安全性{ get：因为可见性和会被缓存等，所以安全性差。 post: 相对好（抓包工具fiddler照样能抓到）。 } 5.参数的数据类型{ get：只支持ASCII字符。 post: 没有限制。 } 6.浏览器刷新{ get：无提示。 post: 提示表单是否重新提交。 } 7.传输操作{ get：一般用于向服务器获取数据。 post: 一般用于向服务器传输数据。 } 什么是报文：交换与传输的数据块。头部：一般由请求方式（get） + url + 传输协议（http） + 协议版本（1.0）等组成 主体：由传输的内容（如cookie和参数等组成） 区别{ get：GET 方法的参数应该放在 url 。 post：POST 方法参数应该放在 body 中。 }
